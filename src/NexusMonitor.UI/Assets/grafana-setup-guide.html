1 What is this?

Nexus Monitor exposes your system metrics through a standard web endpoint called Prometheus. Third-party tools can read that endpoint and store or display your data.

The two tools you'll install are:

  • Telegraf — a lightweight background agent that reads the Nexus Monitor endpoint every few seconds and stores the data in a time-series database.
  • Grafana — a web-based dashboard tool that queries that database and shows you beautiful, real-time charts.
How it fits together
Nexus Monitor → exposes metrics at http://localhost:9182/metrics
Telegraf → scrapes that URL every 10 s and writes to a local database
Grafana → reads the database and renders charts in your browser at http://localhost:3000

Once set up, open http://localhost:3000 in any browser on the same machine to see your live dashboard.

2 Prerequisites

  1. Open Nexus Monitor → Settings → Telemetry.
  2. Toggle Prometheus Endpoint on.
  3. Note the port (default: 9182). Make sure the status indicator reads "Listening on http://…"
Already done
If you clicked "Open Setup Guide" from the Telemetry card, you've already found the right place. Just make sure the toggle is on before continuing.

3 Install Telegraf

Telegraf is a free, open-source agent from InfluxData. Download it from the official site:

influxdata.com — Telegraf downloads

Windows

Download the .zip package (no installer needed). Extract it to a permanent location such as C:\telegraf\. The main executable is telegraf.exe.

Alternatively, if you have Chocolatey:

choco install telegraf

Or with Scoop:

scoop install telegraf

macOS

Install via Homebrew:

brew install telegraf

Or download the macOS binary from the official download page linked above.

Linux (Debian / Ubuntu)

# Add the InfluxData repository key
curl -s https://repos.influxdata.com/influxdata-archive_compat.key | \
  sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/influxdata-archive_compat.gpg

# Add the repository
echo "deb [signed-by=/etc/apt/trusted.gpg.d/influxdata-archive_compat.gpg] \
  https://repos.influxdata.com/debian stable main" | \
  sudo tee /etc/apt/sources.list.d/influxdata.list

sudo apt-get update && sudo apt-get install -y telegraf

Linux (RHEL / CentOS / Fedora)

cat <<EOF | sudo tee /etc/yum.repos.d/influxdata.repo
[influxdata]
name = InfluxData Repository - Stable
baseurl = https://repos.influxdata.com/rhel/\$releasever/\$basearch/stable
enabled = 1
gpgcheck = 1
gpgkey = https://repos.influxdata.com/influxdata-archive_compat.key
EOF

sudo yum install -y telegraf
Tip
You don't need an InfluxDB account or cloud sign-up to use Telegraf locally. It works completely offline.

4 Configure Telegraf

Nexus Monitor generates a ready-to-use Telegraf config snippet. Here's how to use it:

  1. In Nexus Monitor → Settings → Telemetry → Telegraf Integration, click "Save .conf…" and save the file.
  2. Copy the saved .conf file into Telegraf's conf.d directory (Telegraf loads all .conf files from this folder automatically):

    Windows

    C:\Program Files\telegraf\telegraf.d\

    If you extracted the ZIP manually, put it next to telegraf.exe in a telegraf.d\ subfolder.

    macOS / Linux

    /etc/telegraf/telegraf.d/
  3. Telegraf also needs a base configuration. If you don't have one, generate a minimal one: 
    telegraf config > telegraf.conf
    Then edit it to keep only the [agent] section, or use the full generated config as-is.
  4. Start (or restart) the Telegraf service to apply the new config:

    Windows — run as Administrator

    # Install as a Windows service (one-time, run as Admin)
telegraf.exe --service install --config-directory "C:\telegraf\telegraf.d"

# Start the service
net start telegraf

# Or run interactively to see logs:
telegraf.exe --config-directory "C:\telegraf\telegraf.d"

    macOS / Linux (systemd)

    sudo systemctl enable telegraf
sudo systemctl restart telegraf
sudo journalctl -u telegraf -f   # follow logs

    macOS (Homebrew service)

    brew services restart telegraf
Verify Telegraf is working
In a terminal, run telegraf --test --config-directory "/etc/telegraf/telegraf.d" (adjust path for your OS). You should see metric lines printed containing nexus_cpu, nexus_memory, etc. If you see errors, check that the Nexus Monitor Prometheus endpoint is on and that the port in the config matches Settings.

5 Install Grafana

Download Grafana OSS (the free, open-source edition — you don't need Enterprise or a Grafana Cloud account):

grafana.com — Download Grafana OSS

Windows

Download and run the .msi installer. Grafana installs as a Windows service and starts automatically.

Or with Chocolatey:

choco install grafana

macOS

brew install grafana
brew services start grafana

Linux (Debian / Ubuntu)

sudo apt-get install -y adduser libfontconfig1 musl
wget https://dl.grafana.com/oss/release/grafana_11.6.0_amd64.deb
sudo dpkg -i grafana_11.6.0_amd64.deb
sudo systemctl enable --now grafana-server

Check grafana.com for the latest version number.

Once installed, open http://localhost:3000 in your browser. The default login is admin / admin — you'll be prompted to change the password on first login.

Tip
Grafana runs on port 3000 by default. If something else is already using that port, you can change it in /etc/grafana/grafana.ini (Linux/macOS) or C:\Program Files\GrafanaLabs\grafana\conf\defaults.ini (Windows) under [server] → http_port.

6 Add Prometheus Datasource in Grafana

Grafana needs to know where Telegraf is storing the data. Since the Nexus Monitor Telegraf config uses Prometheus remote-write or InfluxDB, you have two options. The simplest path uses Grafana's built-in Prometheus datasource pointed directly at Nexus Monitor:

  1. Open Grafana at http://localhost:3000 and log in.
  2. In the left sidebar, click the Connections icon (or go to Connections → Data sources).
  3. Click "Add new data source".
  4. Search for and select "Prometheus".
  5. Set the Connection URL to: 
    http://localhost:9182
    (Replace 9182 with the port shown in Nexus Monitor Settings if you changed it.)
  6. Leave all other fields at their defaults and scroll down to click "Save & test". You should see a green "Data source is working" message.
Note on Grafana's Prometheus datasource
Grafana's Prometheus datasource queries metrics directly from the /metrics endpoint — no Telegraf required for the datasource connection itself. Telegraf is still useful for storing historical data, but for the dashboard import below you can point Grafana straight at Nexus Monitor.

7 Import the Nexus Monitor Dashboard

Nexus Monitor includes a pre-built Grafana dashboard with panels for CPU, memory, disk, network, GPU, and alerts.

  1. In Nexus Monitor → Settings → Telemetry → Grafana Dashboard, click "Save Dashboard JSON…" and save the file somewhere easy to find (e.g., your Desktop).
  2. In Grafana, click Dashboards in the left sidebar.
  3. Click the "New" button (top right) → "Import".
  4. Click "Upload dashboard JSON file" and select the file you saved in step 1.
  5. In the "Select a Prometheus data source" dropdown, choose the datasource you added in the previous section.
  6. Click "Import". Your dashboard opens immediately with live data.
You're done!
Your Nexus Monitor dashboard is now live. Bookmark http://localhost:3000 and open it any time to see your system stats. Metrics update every 10–15 seconds by default.

8 Troubleshooting

Grafana shows "Data source is not working" or no data
Make sure the Nexus Monitor Prometheus endpoint is toggled on in Settings → Telemetry. Open http://localhost:9182/metrics in your browser — you should see a wall of text starting with # HELP nexus_cpu…. If the page doesn't load, check your firewall (see below).
Browser can't reach http://localhost:9182/metrics
Windows Firewall: Nexus Monitor binds to localhost (127.0.0.1), which is loopback-only — traffic never leaves your machine, so the firewall generally doesn't block it. If you changed the binding to 0.0.0.0 (all interfaces), you may need to allow the port in Windows Firewall.

Port conflict: Another app may already be using port 9182. Change the port in Settings → Telemetry and update the Telegraf config to match.
Telegraf starts but no nexus_* metrics appear in Grafana
Run telegraf --test in a terminal and look for output lines containing nexus_cpu. If you see connection refused errors, the Nexus Monitor endpoint is not reachable from Telegraf — verify the port in the Telegraf .conf file matches the port in Nexus Monitor Settings. Restart Telegraf after making changes.
Grafana dashboard shows "No data" on all panels
After importing, check that the correct Prometheus datasource is selected in the dashboard variables (top of the dashboard → datasource dropdown). Also confirm the time range is set to "Last 15 minutes" or "Last 1 hour" rather than a historic range from before you started Nexus Monitor.
Grafana shows HTTP 403 or authentication errors
The Nexus Monitor /metrics endpoint has no authentication by default — it's accessible to anyone on localhost. If you added auth to the endpoint (via a reverse proxy), configure Grafana's datasource with the matching username/password under the datasource settings.
Telegraf service won't start on Windows
Open Event Viewer → Windows Logs → Application and look for Telegraf errors. Common causes: the .conf file has a syntax error (run telegraf --config … --test to validate), or the service account doesn't have permission to read the config directory. Try running Telegraf interactively as Administrator first to see the raw error output.
Grafana web UI doesn't open at http://localhost:3000
Check that the Grafana server service is running:
  • Windows: Open Services → look for "Grafana" → Start
  • Linux: sudo systemctl status grafana-server
  • macOS: brew services list | grep grafana
If it's running but you can't connect, another app may be on port 3000. Check with netstat -ano | findstr :3000 (Windows) or ss -tlnp | grep 3000 (Linux).

9 Tips & Next Steps

Adjust the scrape interval

By default Telegraf scrapes every 10 seconds. To change it, edit the [agent] block in your base telegraf.conf: 

[agent]
  interval = "5s"       # how often to collect metrics
  flush_interval = "10s" # how often to send to outputs

Send metrics to InfluxDB Cloud (optional)

If you want persistent long-term storage and access from outside your local machine, add an [[outputs.influxdb_v2]] section to your Telegraf config pointing at your InfluxDB Cloud account. Your Grafana instance can then use the InfluxDB datasource instead of Prometheus.

Add Grafana alerting

Grafana can send notifications when metrics cross thresholds — email, Slack, PagerDuty, webhooks, and more. Go to Alerting → Alert rules → New alert rule in Grafana. Nexus Monitor also has its own built-in alert system (Settings → Alerts) for desktop toast notifications.

Monitor remotely

Grafana's web interface is accessible from any device on your local network by browsing to http://<your-PC-IP>:3000. Make sure Windows Firewall allows inbound connections on port 3000 if you want to access it from another machine.

Keep the dashboard up to date

As Nexus Monitor gains new metric types in future releases, an updated dashboard JSON will be available via the same "Save Dashboard JSON…" button in Settings. In Grafana, import the new file and overwrite the existing dashboard to get the latest panels.
